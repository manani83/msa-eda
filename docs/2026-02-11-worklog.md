# 2026-02-11 작업 기록

## 요약
- 로컬 Spring Boot 실행을 컨테이너 실행 방식으로 전환.
- 앱 이미지를 tar 파일로 저장하고, 실행 시 로드하도록 스크립트를 수정.
- `docker-compose.yml`에 앱 서비스 추가.

## 주요 변경 사항
- Docker 이미지 빌드/저장
  - `app/Dockerfile` 추가.
  - 이미지 태그: `hexagonal-app:dev`
  - 저장 경로: `app/docker-images/hexagonal-app_dev.tar`
- 실행 스크립트
  - `scripts/dev-up.sh`에서 `:app:bootRun` 제거.
  - `:app:bootJar` → `docker build` → `docker save` → `docker load` → `docker compose up -d` 순서로 실행.
  - 기존 PID 파일/pgrep 종료 로직은 더 이상 사용하지 않음.
- 종료 스크립트
  - `scripts/dev-down.sh`를 `docker compose down`만 수행하도록 단순화.
- Compose 구성
  - `docker-compose.yml`에 `app` 서비스 추가.
  - MySQL 컨테이너(`mysql`)에 `depends_on` 설정.
  - 앱의 DB 접속을 `mysql` 서비스명으로 연결하도록 환경변수 추가.

## 테스트
- 실행하지 않음.

## 메모
- 앱 이미지는 `app/docker-images/hexagonal-app_dev.tar`로 저장되며, 실행 시 `docker load`로 로드됨.
- 컨테이너 실행은 `docker compose up -d` 한 번으로 MySQL + 앱 모두 기동됨.
